## Typed Subqueries

### IN expression based on single column

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookTable book = BookTable.$;
AuthorTableEx author = AuthorTableEx.$;

List<Book> books = sqlClient
    .createQuery(book)
    .where(
        book.id().in(sqlClient
            .createSubQuery(author) 
            .where(author.firstName().eq("Alex"))
            .select(author.books().id())
        )
    )
    .select(book)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin  
val books = sqlClient
    .createQuery(Book::class) {
        where(
            // highlight-next-line
            table.id valueIn subQuery(Author::class) {
                where(table.firstName eq "Alex")
                select(table.books.id) 
            }
        )
        select(table)    
    }
    .execute()
```

</TabItem>
</Tabs>

The generated SQL is:

```sql
select
    tb_1_.ID,
    tb_1_.NAME,
    tb_1_.EDITION,
    tb_1_.PRICE, 
    tb_1_.STORE_ID
from BOOK as tb_1_
where
    /* highlight-next-line */  
    tb_1_.ID in (
        select
            tb_3_.BOOK_ID
        from AUTHOR as tb_2_
        inner join BOOK_AUTHOR_MAPPING as tb_3_
            on tb_2_.ID = tb_3_.AUTHOR_ID
        where
            tb_2_.FIRST_NAME = ?  
    )
```

### IN expression based on multiple columns

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookTable book = BookTable.$;

List<Book> newestBooks = sqlClient
    .createQuery(book)
    .where(
        Expression.tuple(
            book.name(),
            book.edition()
        ).in(sqlClient
            .createSubQuery(book)
            .groupBy(book.name())
            .select(
                book.name(),
                book.edition().max() 
            )
        )
    )
    .select(book)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin 
val newestBooks = sqlClient
    .createQuery(Book::class) {
        where(
            // highlight-next-line
            tuple(
                table.name,
                table.edition  
            ) valueIn subQuery(Book::class) {
                groupBy(table.name)
                select(
                    table.name, 
                    max(table.edition).asNonNull()
                )
            }
        )
        select(table)
    }
    .execute()
```

</TabItem>
</Tabs>

The generated SQL is:

```sql
select
    tb_1_.ID,
    tb_1_.NAME, 
    tb_1_.EDITION,
    tb_1_.PRICE,
    tb_1_.STORE_ID
from BOOK as tb_1_  
where
    /* highlight-next-line */
    (tb_1_.NAME, tb_1_.EDITION) in (
        select
            tb_2_.NAME,
            max(tb_2_.EDITION)
        from BOOK as tb_2_
        group by tb_2_.NAME 
    )
```

### Treat subquery as simple value

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java  
BookTable book = BookTable.$;

List<Book> newestBooks = sqlClient
    .createQuery(book)
    .where(
        book.price().gt(sqlClient
            .createSubQuery(book)
            .groupBy(book.name())
            .select(
                book
                    .price()
                    .avg()
                    .coalesce(BigDecimal.ZERO)
            )
        )
    )
    .select(book)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin
val books = sqlClient
    .createQuery(Book::class) {
        where(
            // highlight-next-line 
            table.price gt subQuery(Book::class) {
                select(
                    avg(table.price)
                        .coalesce(BigDecimal.ZERO)
                )
            }
        )
        select(table)
    }
    .execute() 
```

</TabItem>
</Tabs>

The generated SQL is:

```sql 
select
    tb_1_.ID,
    tb_1_.NAME,
    tb_1_.EDITION, 
    tb_1_.PRICE,
    tb_1_.STORE_ID
from BOOK as tb_1_   
where
    /* highlight-next-line */
    tb_1_.PRICE > (
        select
            coalesce(avg(tb_2_.PRICE), ?)
        from BOOK as tb_2_  
    )
```

### Use subquery in select and orderBy clauses

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookStoreTable store = BookStoreTable.$;
BookTable book = BookTable.$;

MutableRootQuery<BookStoreTable> query = 
    sqlClient.createQuery(store);
TypedSubQuery<BigDecimal> subQuery =
    sqlClient
        .createSubQuery(book) 
        .where(book.store().eq(store))
        .select(
            book
                .price()
                .avg()
                .coalesce(BigDecimal.ZERO)  
        );
List<Tuple2<BookStore, BigDecimal>> storeAvgPriceTuples =
    query
        
        .orderBy(
            // highlight-next-line
            subQuery.desc() 
        )
        .select(
            store,
            // highlight-next-line
            subQuery
        )
        .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin 
val storeAvgPriceTuples = sqlClient
    .createQuery(BookStore::class) {
        val avgPriceSubQuery = subQuery(Book::class) {
            where(table.store eq parentTable)
            select(avg(table.price))
        }
        
        orderBy(
            // highlight-next-line
            avgPriceSubQuery.desc()
        )
        select(
            table,
            // highlight-next-line
            avgPriceSubQuery
        )
    }
    .execute()
```

</TabItem>
</Tabs>

The generated SQL is:  

```sql
select
    tb_1_.ID,
    tb_1_.NAME,
    tb_1_.WEBSITE,
    (
        /* highlight-next-line */  
        select coalesce(avg(tb_2_.PRICE), ?)
        from BOOK as tb_2_
    )
from BOOK_STORE as tb_1_
order by (  
    /* highlight-next-line */
    select coalesce(avg(tb_2_.PRICE), ?)
    from BOOK as tb_2_ 
) desc
```

### Use EXISTS operator

<Tabs groupId="language">
<TabItem value="java" label="Java">  

```java 
BookTable book = BookTable.$;
AuthorTableEx author = AuthorTableEx.$;

List<Book> books = sqlClient
    .createQuery(book)
    .where(sqlClient
        .createSubQuery(author)
        .where(
            author.books().eq(book),
            author.firstName().eq("Alex") 
        )
        .select(author)
        // highlight-next-line
        .exists()
    )
    .select(book)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin
val books = sqlClient
    .createQuery(Book::class) {
        where(
            // highlight-next-line
            exists(
                subQuery(Author::class) {
                    where(
                        table.books eq parentTable,
                        table.firstName eq "Alex"
                    )  
                    select(table)
                }
            )
        )
        select(table)
    }
    .execute() 
```

</TabItem>
</Tabs>

The generated SQL is:  

```sql
select
    tb_1_.ID,
    tb_1_.NAME,
    tb_1_.EDITION,
    tb_1_.PRICE,
    tb_1_.STORE_ID
from BOOK as tb_1_   
where
    /* highlight-next-line */
    exists (
        select
            1
        from AUTHOR as tb_2_
        inner join BOOK_AUTHOR_MAPPING as tb_3_
            on tb_2_.ID = tb_3_.AUTHOR_ID
        where
            tb_1_.ID = tb_3_.BOOK_ID
        and
            tb_2_.FIRST_NAME = ?  
    )
```

:::info
Note that in the final SQL, the selected column in the subquery is the constant `1`, not what is specified in the Java code.  

This is because the `exists` operator only cares if the subquery can match data, not what columns are selected. Whatever you select in the Java code will be ignored.
:::

## Untyped Subqueries

The last example in previous section is `exists` subquery where whatever is selected in Java code gets ignored.

In that case, why specify the return type for `exists` subqueries in Java code?

Therefore, jimmer-sql supports untyped subqueries (wild subqueries). Unlike normal subqueries, for wild subqueries the final `select()` call is no longer needed, i.e. no return type. 

<Tabs groupId="language">
<TabItem value="java" label="Java">  

```java
BookTable book = BookTable.$; 
AuthorTableEx author = AuthorTableEx.$;

List<Book> books = sqlClient
    .createQuery(book)
    .where(sqlClient
        // highlight-next-line
        .createSubQuery(author)
        .where(
            author.books().eq(book),
            author.firstName().eq("Alex")
        )
        // No select here  
        .exists()
    )
    .select(book)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin
val books = sqlClient
    .createQuery(Book::class) {
        where(
            exists(
                // highlight-next-line
                wildSubQuery(Author::class) {
                    where(
                        table.books eq parentTable, 
                        table.firstName eq "Alex"
                    )
                    // No select here
                }
            ) 
        )
        select(table)
    }
    .execute()  
```

</TabItem>
</Tabs>

The final SQL remains the same:

```sql 
select
    tb_1_.ID,
    tb_1_.NAME,
    tb_1_.EDITION,
    tb_1_.PRICE,
    tb_1_.STORE_ID
from BOOK as tb_1_    
where
    /* highlight-next-line */ 
    exists (
        select
            1
        from AUTHOR as tb_2_
        inner join BOOK_AUTHOR_MAPPING as tb_3_
            on tb_2_.ID = tb_3_.AUTHOR_ID
        where
            tb_1_.ID = tb_3_.BOOK_ID
        and
            tb_2_.FIRST_NAME = ?   
    )
```

:::info
The only value of untyped subqueries is when used with `exists` operator.
:::
