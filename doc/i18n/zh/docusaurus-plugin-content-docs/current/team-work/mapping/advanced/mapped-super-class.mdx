## 基本使用

`org.babyfish.jimmer.sql.MappedSuperclass`用于提供可供实体继承的抽象超类型。

该超类型不并是实体，但可以被多个实体类型继承，从而避免多个实体重复声明相同的属性。

让我们来看一个例子，先定义超类型

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java title="BaseEntity.java"
// highlight-next-line
@MappedSuperclass
public interface BaseEntity {

    LocalDateTime createdTime();

    @ManyToOne
    User createdBy();

    LocalDateTime modifiedTime();

    @ManyToOne
    User modifiedBy();
}
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin title="BaseEntity.kt"
// highlight-next-line
@MappedSuperclass
interface BaseEntity {

    val createdTime: LocalDateTime

    @ManyToOne
    val createdBy: User

    val modifiedTime: LocalDateTime

    @ManyToOne
    val modifiedBy: User
}
```

</TabItem>
</Tabs>

## 多继承

被`MappedSuperclass`修饰的类型支持多继承，其他类型可以从多个`MappedSuperclass`超类型继承。

添加一个新的抽象接口`TenantAware`，所有支持多租户的实体都继承它

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java title="TenantAware.java"
// highlight-next-line
@MappedSuperclass
public interface TenantAware {

    String tenant();
}
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin title="TenantAware.kt"
// highlight-next-line
interface TenantAware {

    val tenant: String
}
```

</TabItem>
</Tabs>

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java title="Book.java"
@Entity
// highlight-next-line
public interface Book extends BaseEntity, TenantAware {

    ...省略其他代码...
}
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin title="Book.kt"
@Entity
// highlight-next-line
interface Book : BaseEntity, TenantAware {

    ...省略其他代码...
}
```

</TabItem>
</Tabs>

修改`Book`，让它不光继承`BaseEntty`，还继承`TenantAware`

:::tip
`@MapperSuperclass`的作用不仅仅是减少重复代码，还可以其他另外两个功能配合使用

-   [全局过滤器](../../query/global-filter)
-   [拦截器](../../mutation/draft-interceptor)

在和它们配合使用时，多继承可以获得良好的灵活性。
:::
